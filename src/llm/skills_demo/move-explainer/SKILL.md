# move-explainer
When to use: the user asks to explain the point of a move.
Steps:
1. Call eval and reason about the move's idea.
2. Use review_move if the move was already played.
3. Explain the purpose in one coaching sentence.
Constraint: ground the explanation in the engine output, not vibes.
