# middlegame-plan
When to use: the user asks for a plan in a middlegame position.
Steps:
1. Call eval and board_state to read the imbalances.
2. Pick the side of the board where you are stronger.
3. State a concrete three-move plan.
Constraint: tie the plan to the actual imbalances on the board.
