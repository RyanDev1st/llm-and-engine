# v1.2 corpus audit — training-readiness (pre-Kaggle)

Data-science + AI-science audit of the committed `v1_2` SFT corpus before spending GPU.
Goal of the corpus: SFT **Gemma 4 E4B** (QLoRA via Unsloth → q4_0 GGUF on a local 4060) into a
**general agentic harness operator** — choose among the prompt-listed skills/tools/plugins and
think to complete a goal in any domain, narrating tool results without computing them — with
**three reasoning behaviors** (fast / think / auto) selected by a prompt signal.

## Status

**PASS — training-ready. GO for Kaggle.** Coherent contract, correct loss masking, zero true
duplicate examples, grounded compute slice, fits the 1664 seq ceiling, and the three reasoning
modes are quantitatively distinct. Two caveats below are *measurement/known-design* facts, not
data defects. Architecture is sound and additive; no refactor needed for Stage 0.

## Scope

Committed split: `data/sft/v1_2_train.jsonl.gz` (**73,049**) + `v1_2_val.jsonl.gz` (**2,015**),
29 slices. Audited: structure/contract coherence, loss mask, dedup, leakage, mix, final
diversity, reasoning-mode integrity, and the new `V1_R_compute_grounding` (Stage-0 python tool).

## Evidence

Commands: `python scripts/final_corpus_audit.py` (GATE), plus targeted analysis over the gz.

### Structure & coherence — PASS
- Per-row validation: **0 / 75,064** failures (`validate_row` over every accepted row).
- One renderer train==serve (`system_prompt.build_system` from the per-row envelope).
- **Loss mask** (`data_pipeline.tokenize_with_assistant_mask`): only assistant turns trained
  (user/tool/system → `-100`); tool results remapped to **masked** user turns (read as context,
  never trained — `chat_format.remap_tool_messages`); `GROUND_WEIGHT=5×` on fact tokens (eval
  numbers, SAN) so fabrication is penalized. Correct for grounding.
- Template fallback **0**, tool-result survival **100%** (every tool-bearing row shows
  `<tool_result>` after remap — the bug that blinded the first adapter is absent).
- **True full-example dups (system+messages): 0.** The gate's `train_exact_dups` (4,571) is a
  *messages-only* hash; once the per-row system prompt (skills/tools sampling) is included,
  every one of 73,049 examples is unique. No memorization fuel.
- Token length: **max 1655 < 1664** ceiling (p50 1291, p99 1551, 0 over).

### Reasoning-mode integrity — PASS (the fast/think/auto goal, quantified)
Mean `<think>`-per-assistant-step by mode — three genuinely distinct behaviors:

| mode | n (train) | mean think-ratio | rows w/ any think | rows think-every-step |
|---|---|---|---|---|
| **fast** | 25,898 | **0.000** | 0.0% | 0.0% |
| **auto** | 28,884 | **0.575** | 100% | 15.1% |
| **think** | 18,267 | **0.969** | 100% | 91.2% |

- fast NEVER thinks; think thinks on ~every step (the 9% gap = the `execute`-kind rote chain
  steps that never think by design); **auto is interleaved** — always reasons at least once but
  only on ~half the steps (hard decisions), clearly distinct from both.
- **All three modes appear in every slice** → the prompt SIGNAL drives the behavior, not the
  slice. Gate integrity: `fast_with_think=0`, `think_without_think=0`.

### Mix — PASS
chess **24.1%** / general **74.6%** / compute **1.4%** — matches the ~75/25 general-first design.

### V1_R compute-grounding (Stage-0 python tool) — clean
- **990 train / 105 val** (bumped from 30→80 base ≈ ~28 examples per (3 modes × 12 families)
  cell — robust-learning threshold for a 4B, vs ~10 before).
- `python` tool in **only** its 990 manifests (0 leakage → additive, per-row manifest works).
- Grounding violations **0**; every two-decimal number in a final comes from the tool's stdout.
- **Held-out val 94/105 = 89.5%** novel prompts (random numeric params) — the only slice whose
  val is a true generalization probe; ideal for the Stage-0 question.
- Per-row seq max **1469 < 1664** (count bump doesn't change per-row length).

### Two caveats (measurement / known-design, NOT defects)
1. **Val is a same-distribution loss probe, not a generalization benchmark.**
   `val_first_prompt_in_train = 1,854 / 2,015 ≈ 92%` (structural: prompts come from small fixed
   pools; the big templated slices are ~100% overlap). `val_exact_row` and `val_final_text`
   overlap are both **0**. → Do not early-stop on val loss or cite it as generalization; use
   `eval_routing.py` (held-out skills_demo) + live serve. **V1_R is the exception** (held-out).
2. **Content repetition in lesson slices** (e.g. V1_Q 10 finals / 2,486 rows). Repeated *final
   sentences*, each in a *unique full example* (different skills context) — bounded by design
   for decline/statement-contract behavior; 0 true dups, so not memorization.

### Architecture verdict
Sound + coherent for Stage 0; **extension, not refactor, for Stage 1/2**: add optional
goal/plan/box envelope fields, a 4th `plan` behavior, and a slice→renderer registry when slices
multiply. One forward principle: **parametrize prompts (the V1_R way) instead of enumerating
fixed phrasings** — this is what gives V1_R held-out val and high diversity, and is required for
compound-goal diversity in Stage 1. Do not retrofit old slices (the "don't invent phrasings"
rule still holds).
