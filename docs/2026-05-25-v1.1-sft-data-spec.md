## Status

V1.1 SFT data-quality spec. Supersedes the v1 spec. Driven by the v1.1 implementation plan in docs/superpowers/plans/.

## Scope

Mirror Anthropic's harness exactly: enabled user skills and plugin skills merge into one flat skill index per turn; tools live inside plugins; tool manifest is the full enabled list per turn; hook gates run as static `applies_when` predicates plus a runtime guard before exec. Train a universal agent that reads the index, picks by description, loads only selected skill bodies, follows them strictly, and answers in plain English with no XML.

## Evidence

- Existing chess routing corpus design in `chess_assistant_sft_dataset_spec_v3.md`.
- v0.1 prototype runtime adds multi-tool loop, skill loader, board_state on demand.
- User direction: mirror Anthropic harness, numerous + slice-quality dataset, include adversarial-synthetic names for universality, hook gates as `applies_when` + runtime guard (no self-check turn).

## Row Contract

Same fields as v1 plus: `kind`, `plugin_context`, `position_fen`, `stockfish_truth`. Tools in `tool_manifest` may declare `applies_when`; renderer must not call tools whose predicate fails for the position.

## Acceptance Rules

All v1 rules plus:

- `applies_when_respected`: no tool call is emitted for a tool whose `applies_when` is false for the current position.
- `plugin_only_tools`: every tool call resolves to a tool inside an enabled plugin in `plugin_context`.
- `skill_body_strict`: when a skill body lists explicit numbered steps or commands, the assistant's tool chain follows that order.
- `engine_grounded`: chess facts in the final answer agree (sign + magnitude class) with Stockfish output for that position.
