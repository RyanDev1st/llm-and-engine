   - Does each accepted row load `chess-coach` before using chess tools?
   - Are engine claims grounded in `board_state`, `eval`, `best_move`, `review_move`, or `threats` tool outputs?
   - Do illegal/refusal examples avoid executing moves without legal tool evidence?
   - Do rejected rows cover real failure modes: disabled plugin tools, false install claims, absent skills, invalid args, duplicate calls, and XML leakage?
   - Are marketplace/plugin-source rows training dynamic availability rather than memorized tool names?
   - Are final answers natural and XML-free while still matching tool results?
   - Are prompt styles varied enough that repeated slice prompts do not dominate?
   - Are Stockfish-derived best moves/evals represented as data, not unsupported claims?
